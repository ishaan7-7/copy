Introduction & Why MQTT

Vehicle telemetry is the automated collection, transmission, and analysis of data from a moving vehicle to a remote system — covering GPS position, engine diagnostics, driver behaviour scoring, and predictive maintenance signals.

MQTT (Message Queuing Telemetry Transport) was originally designed for pipeline monitoring over satellite links — constrained networks with low bandwidth and high latency. These same properties make it the gold standard for vehicle telemetry over 2G/3G/4G/5G cellular and NB-IoT links.

ℹ️
Key Design Insight MQTT's minimum fixed header is only 2 bytes, versus ~500 bytes for the smallest HTTP request. On a congested 2G network or narrowband IoT link, this difference directly translates to reliability and cost.

Raw Vehicle Data

Before any software touches the data, sensors generate raw analog or digital signals. Understanding this layer is critical for designing the ETU's data collection pipeline. There are four primary sources.

2.1 CAN Bus — Controller Area Network

The CAN bus is the vehicle's internal nervous system. Modern vehicles have 2–5 CAN buses (powertrain, chassis, body, ADAS, infotainment). Every ECU broadcasts data frames on the bus at up to 1 Mbit/s.

📋
Frame Format A CAN 2.0A standard frame contains an 11-bit message ID (identifying which ECU and signal), a DLC field (0–8 data bytes), and 8 data bytes. The ETU reads these frames from a CAN controller IC (e.g., MCP2515) over SPI. 
// Raw CAN frame — Engine RPM (PID 0x0C9)
ID:    0x0C9  (201 decimal)
DLC:   8      (8 data bytes)
Data:  0F A0 00 00 00 00 00 00

// Decode using DBC formula: RPM = (Byte0 × 256 + Byte1) / 4
     = (0x0F × 256 + 0xA0) / 4
     = (15 × 256 + 160)    / 4
     = 4000 / 4
     = 1000 RPM

// Vehicle Speed frame (PID 0x0D0)
ID:    0x0D0
Data:  00 64 00 00 00 00 00 00
       Speed = Byte1 = 0x64 = 100 km/h

// Coolant Temperature (PID 0x110) — formula: A - 40
Data:  5A 00 00 00 00 00 00 00
       Temp = 0x5A - 40 = 90 - 40 = 50 °C

For diagnostic data, the ETU uses the OBD-II protocol — a standardised request/response layer running on top of CAN: 

// Request — Mode 01, PID 0x0C (Engine RPM)
7DF  02  01  0C  00 00 00 00 00
      |   |   └── PID: Engine RPM
      |   └────── Mode 01 = live data
      └────────── 02 = 2 data bytes follow

// Response from Engine ECU
7E8  04  41  0C  0F A0  00 00 00
          |   |   └──┴── 0x0FA0 → RPM = 4000/4 = 1000 RPM
          |   └──────── PID echo 0x0C
          └──────────── Mode response = 0x41 (0x01 + 0x40)

2.3 IMU — Raw I²C Register Reads

An Inertial Measurement Unit (e.g., MPU-6050) measures acceleration and rotation at 100–200 Hz over I²C. Raw values are 16-bit signed integers that require scaling. 

// MPU-6050 at I²C 0x68 — Accelerometer registers 0x3B–0x40
Raw bytes:   FF E0   00 20   3F C0
              ─────   ─────   ─────
              AccX    AccY    AccZ (16-bit signed, big-endian)

AccX_raw = 0xFFE0 = -32   →  AccX_g = -32  / 16384 = -0.00195 g
AccY_raw = 0x0020 = +32   →  AccY_g = +32  / 16384 = +0.00195 g
AccZ_raw = 0x3FC0 = +16320 → AccZ_g = 16320 / 16384 = +0.996 g  ← gravity

// Gyroscope registers 0x43–0x48
Raw bytes:   00 14   FF FC   00 08

GyroX = 0x0014 =  20  →  20  / 131 =  0.153 °/s
GyroY = 0xFFFC =  -4  →  -4  / 131 = -0.031 °/s
GyroZ = 0x0008 =   8  →   8  / 131 =  0.061 °/s

Edge Telematic Unit (ETU)

The ETU is the hardware + software device installed in the vehicle — the bridge between raw sensor signals and the cloud. It performs seven distinct processing stages before data ever reaches MQTT.

3.2 The 7-Step ETU Processing Pipeline

1
Signal Acquisition INTERRUPT-DRIVEN
Dedicated threads read each interface at its native rate. CAN bus is interrupt-driven (up to 1 Mbit/s), GPS at 1–10 Hz, IMU at 100–200 Hz, and ADC at 1 Hz. Data is pushed into ring buffers to avoid blocking the acquisition thread.
2
Decoding & Parsing DBC / PID TABLES
Raw CAN bytes are decoded using DBC signal definitions: physical = raw × factor + offset. NMEA sentences are parsed with a regex-based parser. IMU raw counts are divided by sensitivity constants (16384 LSB/g for ±2g range).
3
Filtering & Validation KALMAN / EMA
A 1D Kalman filter smooths GPS position. An Exponential Moving Average (α=0.1) removes vibration from IMU readings. Range checks and rate-of-change limits reject spikes (e.g., RPM cannot jump 1000 in 100 ms).
4
Aggregation & Enrichment 5-SECOND WINDOWS
50 raw samples (at 10 Hz) are aggregated into mean/max/min statistics per 5-second window. The ETU then enriches with device metadata, geofence zone, road speed limit, derived fuel economy, and ignition state.
5
Event Detection Engine REAL-TIME
Every raw sample is evaluated against threshold rules. Triggered events (hard brake, overspeed, collision, ignition) are published immediately at QoS 2 — bypassing the aggregation window entirely.
6
Local Storage & Buffering SQLITE
If the MQTT publish fails (network outage), messages are stored in a local SQLite database as a FIFO queue. On reconnection, the ETU flushes buffered messages in chronological order — guaranteeing zero data loss during tunnels and dead zones.
7
MQTT Client & Transmission TLS 1.3
The Paho MQTT client connects over TLS with mutual X.509 authentication, a persistent session (clean_session=false), and a Last Will & Testament message. Telemetry is serialised as msgpack (~40% smaller than JSON) before publishing.

float decode_signal(const DBC_Signal* sig, const uint8_t* data) {
    uint64_t raw     = extract_bits(data, sig->start_bit, sig->length);
    float    physical = (float)raw * sig->factor + sig->offset;
    return   clamp(physical, sig->min_val, sig->max_val);
}

Step 7 in Detail — MQTT Client (Python/Paho)

import paho.mqtt.client as mqtt, ssl, json, msgpack

class ETUMqttClient:
    def __init__(self, config):
        self.client = mqtt.Client(client_id=config["device_id"], clean_session=False, protocol=mqtt.MQTTv5)
        self.client.tls_set(ca_certs="/certs/ca.crt", certfile="/certs/device.crt", keyfile="/certs/device.key", tls_version=ssl.PROTOCOL_TLS_CLIENT)
        self.client.will_set(
            topic=f"fleet/{config['fleet_id']}/{config['device_id']}/status",
            payload=json.dumps({"status": "offline", "reason": "unexpected_disconnect"}),
            qos=1, retain=True
        )

    def publish_telemetry(self, payload_dict):
        topic         = f"fleet/{fleet_id}/{device_id}/telemetry"
        payload_bytes = msgpack.packb(payload_dict, use_bin_type=True)
        result        = self.client.publish(topic, payload_bytes, qos=1)
        if result.rc != mqtt.MQTT_ERR_SUCCESS:
            self._store_in_sqlite_buffer(topic, payload_bytes)

    def publish_event(self, event):
        topic = f"fleet/{fleet_id}/{device_id}/events/{event['event_type'].lower()}"
        self.client.publish(topic, json.dumps(event).encode(), qos=2)

MQTT Protocol Deep Dive

4.1 Packet Structure & Wire Format

💡
Every MQTT packet starts with a 2-byte fixed header: 4 bits for packet type, 4 bits for flags, then a variable-length remaining-length field. The minimum valid MQTT packet is 2 bytes — that's the entire PINGREQ. 
Fixed Header layout:
┌──────────────────────────────────────────────────┐
│ Byte 1: [ PacketType:4 ][ Flags:4 ]              │
│ Byte 2+: Remaining Length (1–4 bytes, 7-bit enc.)│
└──────────────────────────────────────────────────┘

Packet Types:
  0001=CONNECT   0010=CONNACK   0011=PUBLISH   0100=PUBACK
  0101=PUBREC    0110=PUBREL    0111=PUBCOMP   1000=SUBSCRIBE
  1001=SUBACK    1100=PINGREQ   1101=PINGRESP  1110=DISCONNECT

PUBLISH packet annotated (QoS 1):
Offset  Hex    Meaning
──────  ─────  ──────────────────────────────────────────────
0       32     Fixed header: 0011=PUBLISH, 0010=QoS1
1       37     Remaining length = 55 bytes
2–3     00 23  Topic string length = 35 characters
4–38    ...    Topic: "fleet/FLEET-001/ETU-001/telemetry"
39–40   00 01  Packet Identifier = 1  (only for QoS 1 & 2)
41–55   ...    Payload (msgpack binary)

5.4 Retained Messages & Last Known State

The broker stores one retained message per topic. A new subscriber immediately receives the most recent value without waiting for the next publish cycle — critical for device status and fault codes.

.../status ✅ Yes New subscribers must immediately know if vehicle is online/offline
.../telemetry ❌ No High-frequency; stale data shown as current would be misleading
.../events/+ ❌ No Events are point-in-time; replaying them to new subscribers is wrong

Quality of Service (QoS) Levels

MQTT defines three delivery guarantee levels. Choosing the right QoS is a trade-off between reliability and message overhead.

QoS 0

At Most Once

Fire and forget. No acknowledgement, no retry. Fastest and smallest overhead. Message may be lost if network drops at the wrong moment.

Use for: GPS position (1 Hz), 100 Hz IMU stream on good LTE
QoS 1

At Least Once

Publisher retransmits until PUBACK received. Message is guaranteed to arrive, but may arrive more than once. Subscribers must deduplicate.

Use for: 5-second telemetry windows, DTC fault codes
QoS 2

Exactly Once

4-way handshake (PUBLISH → PUBREC → PUBREL → PUBCOMP). Highest overhead but guaranteed exactly-once delivery.

Use for: Hard brake, collision alert, ignition on/off, FOTA ACK
Data Type Frequency QoS Reason
GPS live position 1 Hz 0 Next update makes this one stale; loss acceptable
Aggregated telemetry Every 5–30 s 1 Occasional duplicate is tolerable; no loss desired
Driver behaviour events On trigger 2 Must not be missed or doubled in scoring calculations
Ignition on / off On trigger 2 Drives billing and trip boundary calculations
DTC fault codes On trigger 1 Important, slight duplicate acceptable
Collision alert On trigger 2 Emergency — cannot be lost or duplicated
FOTA acknowledgement On trigger 2 Firmware update confirmation must be exactly once

8.2 Payload Signing for High-Value Events

import hmac, hashlib, base64

def sign_event(payload_bytes, device_id, epoch_ms, secret):
    msg = f"{device_id}:{epoch_ms}:".encode() + payload_bytes
    sig = hmac.new(secret, msg, hashlib.sha256).digest()
    return base64.b64encode(sig).decode()

event["_sig"] = sign_event(json.dumps(event).encode(), event["device_id"], event["epoch_ms"], DEVICE_SECRET)

Term Definition
ETU Edge Telematic Unit — in-vehicle hardware/software that collects, processes, and transmits vehicle data to the cloud.
CAN Bus Controller Area Network — the wiring and protocol allowing ECUs inside a vehicle to communicate without a central host.
ECU Electronic Control Unit — a microcomputer controlling one vehicle subsystem (engine, ABS, transmission, airbags, etc.).
OBD-II On-Board Diagnostics II — standardised interface for reading vehicle diagnostic and live sensor data via a 16-pin port.
PID Parameter ID — a code used to request a specific data point (e.g., 0x0C = Engine RPM) from the OBD-II interface.
DBC File Database CAN file — a text format describing how to decode raw CAN bus bytes into named, scaled engineering values.
NMEA 0183 National Marine Electronics Association standard — defines the ASCII sentence format used by GPS/GNSS receivers.
MQTT Broker The server that receives PUBLISH packets and routes them to all matching SUBSCRIBE clients.
QoS Quality of Service — the delivery guarantee level (0=fire-and-forget, 1=at-least-once, 2=exactly-once).
Retained Message A message stored by the broker and delivered instantly to any new subscriber — like a "last known value" cache per topic.
LWT Last Will & Testament — a message the broker publishes on behalf of a client that disconnects unexpectedly (ungracefully).
Clean Session If true, broker discards all session state on disconnect. If false (persistent), broker holds QoS 1/2 queues for the client.
DUP Flag Duplicate flag — set on a retransmitted PUBLISH packet so subscribers can detect and discard duplicates.
HDOP Horizontal Dilution of Precision — GPS accuracy indicator. Under 1.0 = excellent; 1–2 = good; above 5 = poor.
IMU Inertial Measurement Unit — sensor measuring linear acceleration (g-force) and angular velocity (gyroscope).
DTC Diagnostic Trouble Code — standardised fault code (e.g., P0300 = random cylinder misfire) stored by the vehicle ECU.

Sparkplug B An MQTT-based specification for industrial IoT — adds structured birth/death certificates, sequence numbers, and protobuf encoding.
Msgpack Binary serialisation format — semantically equivalent to JSON but ~40% smaller, reducing MQTT payload bandwidth and latency.

ACL Access Control List — broker-enforced rules defining which MQTT clients may publish or subscribe to which topic patterns.

0. Introduction & Architecture

OVERVIEW
Modern vehicles contain 50–150 Electronic Control Units (ECUs) interconnected by in-vehicle networks. These ECUs continuously generate telemetry signals — real-time measurements of physical, electrical, and logical states — which are used for closed-loop control, diagnostics, safety, and remote monitoring.

Network Protocols:
🟦 CAN (ISO 11898) — Powertrain, Chassis; 125 kbps–1 Mbps 🟨 LIN — Low-speed body functions; 20 kbps 🟥 FlexRay — Safety-critical (steer/brake); 10 Mbps 🟩 Automotive Ethernet — ADAS, Infotainment; 100M–1G bps ⬜ MOST — Multimedia; 25–150 Mbps
Signal Criticality Tiers:
CRITICAL Safety/drivability — loss may cause accident or severe damage MODERATE Emission/performance — fault degrades function LOW Comfort/informational — loss is inconvenient, not dangerous DIAGNOSTIC Used primarily for workshop fault diagnosis

1. Engine Control Module (ECM / ECU / PCM)

POWERTRAIN
The ECM is the most signal-dense ECU. It manages fuel injection, ignition timing, air/fuel ratio, variable valve timing, turbocharger boost, and exhaust aftertreatment. It broadcasts signals on the high-speed CAN powertrain bus (500 kbps) and exposes ~200+ OBD-II PIDs.
Signal Name OBD PID / J1939 SPN Unit Typical Range Criticality Significance
Engine_Speed (RPM) PID 0x0C / SPN 190 rpm 0 – 8000 CRITICAL Primary indicator of engine operating state. Used by TCM for shift logic, by ECM for fuel/ignition maps. Idle: 650–850 rpm; Redline: 5500–8000 rpm depending on engine type.
Engine_Load (%) PID 0x04 % 0 – 100 CRITICAL Calculated load based on actual vs. max air mass. Drives fuel injection pulse width and ignition advance maps. High load at low RPM indicates potential knock conditions.
Coolant_Temperature PID 0x05 / SPN 110 °C -40 – 215 CRITICAL Controls cold-start enrichment, fan activation, torque limiting. Normal op: 85–105°C. Above 115°C triggers warning; above 120°C may trigger limp-home.
Intake_Air_Temperature (IAT) PID 0x0F / SPN 105 °C -40 – 120 MODERATE Used to calculate air density for fuel trim. High IAT reduces volumetric efficiency and advances timing toward knock boundary. Critical for turbocharged engines.
Mass_Air_Flow (MAF) PID 0x10 / SPN 132 g/s 0 – 600 CRITICAL Direct measure of air entering engine. Primary input to speed-density fueling. Failure causes rich/lean misfire, stall. Typical idle: 2–7 g/s; WOT: up to 600 g/s on large engines.
Manifold_Absolute_Pressure (MAP) PID 0x0B / SPN 102 kPa 10 – 300 CRITICAL Backup to MAF on speed-density systems; primary on MAP-only systems. Idle vacuum: 30–50 kPa; WOT naturally aspirated: ~101 kPa; turbocharged at boost: 150–250 kPa.
Throttle_Position (TPS) PID 0x11 / SPN 91 % 0 – 100 CRITICAL Driver demand signal on fly-by-wire systems. Dual/redundant sensors for ASIL-D compliance. Discrepancy between pedal demand and throttle angle triggers torque reduction.
Accelerator_Pedal_Position (APP) PID 0x49 / SPN 91 % 0 – 100 CRITICAL Raw driver torque demand. Typically has 2–3 redundant sensors (APP1, APP2, APP3). Used to calculate target torque before traction control arbitration.
Short_Term_Fuel_Trim (STFT) PID 0x06 % -25 – +25 MODERATE Instantaneous closed-loop fuel correction based on O2 sensor feedback. Values >±10% indicate sensor drift, vacuum leak, or injector fault.
Long_Term_Fuel_Trim (LTFT) PID 0x07 % -25 – +25 MODERATE Learned baseline fuel correction stored in NVRAM. Persistent lean LTFT (>+10%) indicates air leak; rich LTFT (<-10%) indicates high fuel pressure or leaking injector.
O2_Sensor_Upstream (Bank1) PID 0x14 / SPN 3226 V / λ 0.1 – 0.9 V CRITICAL Pre-cat lambda sensor. Oscillates 0.1–0.9 V at ~2 Hz in closed loop (stoichiometric). Flat signal = sensor failure. Drives STFT corrections.
O2_Sensor_Downstream (Bank1) PID 0x18 V 0.5 – 0.8 V MODERATE Post-cat sensor monitors catalytic converter efficiency. If it oscillates like upstream sensor, catalyst is degraded. Stable high voltage = good catalyst.
Fuel_Rail_Pressure PID 0x23 / SPN 94 kPa 200 – 20000 CRITICAL Port injection: 250–450 kPa. GDI: 3000–20000 kPa. Low pressure causes lean misfire; excessive pressure causes rich condition and injector damage.
Ignition_Timing_Advance PID 0x0E °BTDC -64 – +64 CRITICAL Degrees of crank rotation before TDC at which spark fires. Retarded timing reduces power and increases exhaust temperature. Knock-induced retard visible here.
Knock_Sensor_Signal SPN 3563 mV/count 0 – 4095 CRITICAL Raw vibration amplitude at ~6–8 kHz knock frequency. ECM applies window filtering; persistent high counts retard timing cylinder-specifically. Key for performance/ethanol tuning.
Cam_Position (VVT_Angle) SPN 637 °CA 0 – 50 MODERATE Actual vs. target camshaft phasing angle. Error >5° triggers DTC. Controls intake/exhaust overlap for torque, fuel economy, and emissions across RPM range.
Crank_Position_Signal SPN 636 ° 0 – 360 CRITICAL Absolute crankshaft position from 60-2 or 36-1 reluctor wheel. Loss = engine stall. Used for injection phasing, ignition timing, and misfire detection.
EGR_Position (%) PID 0x2C / SPN 27 % 0 – 100 MODERATE Exhaust Gas Recirculation valve opening. Reduces NOx by lowering combustion temperature. Critical for diesel emissions compliance. Stuck-open causes rough idle; stuck-closed fails emissions.
Boost_Pressure (Turbo) PID 0x70 / SPN 102 kPa (gauge) 0 – 200 CRITICAL Turbocharger boost above atmospheric. Over-boost triggers fuel cut to protect engine. Under-boost at WOT indicates wastegate, VGT, or intercooler leak.
Wastegate_Duty_Cycle SPN 1131 % 0 – 100 MODERATE Solenoid PWM signal controlling boost pressure. High duty cycle = closed wastegate = more boost. Used with boost sensor in closed-loop boost control.
Fuel_System_Status PID 0x03 enum OL/CL/OL-fault MODERATE Open Loop (cold start / WOT enrichment) vs. Closed Loop (normal O2 feedback). OL-fault indicates O2 sensor failure.
Evap_System_Monitor PID 0x1C Pa -500 – +500 DIAGNOSTIC Evaporative emission canister purge system pressure. Used during EVAP leak test (0.020"/0.040" orifice). Failure sets P0440–P0457.
Oil_Temperature SPN 175 °C -40 – 180 MODERATE Engine oil temperature. Critical for variable oil pump control and VVT oil pressure. Full oil film strength achieved at ~80°C. Above 150°C degrades oil film lubrication.
Oil_Pressure SPN 100 kPa 0 – 700 CRITICAL Lubrication circuit pressure. Low oil pressure at idle (<70 kPa) triggers engine shutdown to prevent catastrophic bearing failure. Normal: 200–400 kPa at operating RPM.
Fuel_Injection_Pulse_Width SPN 1injection ms 0.5 – 20 DIAGNOSTIC Duration injectors are energized per cycle. Reflects actual fueling command; combined with RPM gives fuel consumption estimation. Critical for injector flow-rate matching.
Misfire_Count (per cylinder) PID 0x4C–0x4F count 0 – 65535 CRITICAL Individual cylinder misfire events detected via crankshaft acceleration dropout. High misfire rate on one cylinder = injector, coil, or mechanical fault. Misfire catalyst damage threshold triggers P0300.
Catalyst_Temperature PID 0x3C / SPN 3246 °C 200 – 1000 CRITICAL Three-way catalyst brick temperature. Light-off temp: ~250°C. Optimal: 400–800°C. Sustained above 900°C causes substrate melt. Misfire dumps raw fuel, triggering thermal overrun.
Exhaust_Gas_Temperature (EGT) SPN 173 °C 100 – 1000 CRITICAL Critical for diesel DPF/SCR and turbocharged petrol engines. Above 950°C may damage turbo. Used for DPF active regeneration control and SCR dosing strategy.
DPF_Differential_Pressure SPN 3251 kPa 0 – 50 MODERATE Pressure drop across diesel particulate filter. As soot accumulates, pressure rises. Above regeneration threshold triggers active DPF regen. Very high values indicate blocked DPF.
NOx_Sensor_Upstream / Downstream SPN 3226/3227 ppm 0 – 3000 MODERATE NOx concentration in exhaust, pre and post SCR catalyst. Efficiency = (upstream - downstream)/upstream. Regulatory compliance monitoring. Triggers OBD readiness fail if efficiency <70%.
AdBlue_Level (DEF) SPN 1761 % 0 – 100 MODERATE Diesel Exhaust Fluid tank level. Below 10% triggers warning; below 2.5% may limit vehicle speed under Euro 6 regulations. Quality also monitored via refractive index sensor.

2. Transmission Control Module (TCM / TCU)

POWERTRAIN
Manages automatic, dual-clutch (DCT), or continuously variable (CVT) transmissions. Receives torque request from ECM, wheel speed from ABS, and driver input from the gear selector. Communicates on the powertrain CAN bus.
Signal Name SPN / PID Unit Typical Range Criticality Significance
Transmission_Output_Speed SPN 191 rpm 0 – 8000 CRITICAL Propshaft/output shaft speed. Combined with input speed gives gear ratio. Discrepancy indicates slipping clutch pack or torque converter lockup failure.
Transmission_Input_Speed SPN 161 rpm 0 – 8000 CRITICAL Speed at turbine shaft (AT) or clutch input. Ratio to output determines current gear. Used to detect torque converter slip and clutch engagement quality.
Transmission_Fluid_Temperature SPN 177 / PID 0x5F °C -40 – 160 CRITICAL Controls shift scheduling (colder = firmer, hotter = softer shifts to protect clutches). Above 130°C triggers torque reduction; above 150°C triggers limp-home (single gear).
Current_Gear SPN 523 enum P/R/N/1–9 CRITICAL Actual engaged gear vs. driver-selected gear. Mismatch indicates shift actuator fault. Broadcast to instrument cluster for gear display and to ECM for gear-dependent torque maps.
Target_Gear SPN 524 enum 1–9 MODERATE TCM's commanded next gear during shift. Prolonged mismatch between target and current gear triggers shift fault DTC.
Torque_Converter_Lockup_State SPN 573 bool 0/1 MODERATE Indicates whether TCC (Torque Converter Clutch) is locked. Unlocked above threshold speed wastes fuel. Slip during lockup indicates worn TCC friction material.
TCC_Slip_Speed SPN – rpm 0 – 300 MODERATE Difference between impeller and turbine speed in torque converter. Normal slip in drive: <100 rpm; controlled slip strategy: 20–80 rpm for fuel economy. High slip = worn TCC.
Line_Pressure (Hydraulic) SPN – kPa 400 – 2000 CRITICAL Transmission main hydraulic line pressure. Too low = clutch slip and damage; too high = harsh shifts and solenoid wear. Controlled by EPC (Electronic Pressure Control) solenoid duty cycle.
Shift_Solenoid_State (SS1–SS6) SPN – bool/PWM % 0–100% DIAGNOSTIC Individual shift solenoid commanded states determine hydraulic routing to select gear clutch packs. Solenoid stuck energized or de-energized prevents specific gear engagement.
Range_Sensor_Position SPN 1875 enum P/R/N/D/L CRITICAL Gear selector position from PRNDL sensor / TR sensor. Used to enable starter motor only in P/N, reverse lights in R, and shift inhibit logic. Fault prevents engine start or causes unintended movement.
Clutch_Pack_Temperature (DCT) – °C -40 – 250 CRITICAL DCT-specific. Wet clutch oil temperature estimated or measured. Excessive launch/slip heat above 200°C triggers cooling mode (inhibits launch control, forces cooling cycle).
CVT_Belt_Ratio – ratio 0.4 – 2.8 MODERATE Pulley ratio in CVT. Belt slip monitoring via primary/secondary speed ratio vs. commanded. Slip damages steel push-belt and causes progressive CVT failure.

3. ABS / ESC / VDC Module (Brake Control)

CHASSIS & SAFETY
The ABS/ESC module is the primary chassis safety ECU. It reads wheel speeds, lateral acceleration, yaw rate, and steering angle to modulate brake pressure individually per wheel. Communicates on high-speed CAN; safety signals often also on FlexRay in premium vehicles.
Signal Name SPN / ISO Unit Typical Range Criticality Significance
Wheel_Speed_FL/FR/RL/RR SPN 904–907 km/h or rpm 0 – 300 km/h CRITICAL Independent wheel speed from each corner's tone wheel sensor (passive VR or active Hall). Foundation for ABS, TCS, ESC, and vehicle speed calculation. Disagreement between wheels triggers stability intervention.
Vehicle_Speed (ABS-derived) SPN 84 / PID 0x0D km/h 0 – 300 CRITICAL Calculated from minimum/average wheel speed. Reference for speedometer, transmission shift schedule, cruise control, and collision avoidance. Shared to all ECUs via CAN gateway.
Yaw_Rate ISO 11992 °/s -100 – +100 CRITICAL Rotational velocity around vertical axis. Core ESC input. Compares actual yaw to driver-intended yaw (from steering angle + speed). Excess yaw = oversteer; deficit = understeer. Triggers brake intervention on individual wheels.
Lateral_Acceleration (Ay) ISO 11992 m/s² or g -15 – +15 m/s² CRITICAL Side acceleration from MEMS sensor in ESC module. Combined with yaw rate for full 6-DOF vehicle dynamics model. Rollover threshold detection in RSC (Roll Stability Control) uses this signal.
Longitudinal_Acceleration (Ax) – m/s² -15 – +15 CRITICAL Forward/rearward acceleration. Used for hill-hold, grade estimation, and collision pre-charge of brakes. Also fed to ADAS for adaptive cruise control response.
Steering_Angle SPN 1807 ° (relative) -720 – +720 CRITICAL Multi-turn steering wheel angle from SAS (Steering Angle Sensor). Must be calibrated after wheel alignment. ESC uses this as driver intended path reference. Also used by lane-keep assist and rear-wheel steering.
Brake_Pressure_Master_Cylinder SPN 521 bar or kPa 0 – 200 bar CRITICAL Driver brake pedal force translated to hydraulic pressure. Used by ESC for brake proportioning and by ADAS for autonomous emergency braking (AEB). Dual-circuit pressure monitoring for brake failure detection.
Brake_Switch_Status SPN 597 bool 0/1 CRITICAL Binary brake pedal depressed signal. Activates brake lights, disengages cruise control, enables EBD (Electronic Brake-force Distribution) entry. Redundant dual switch required for AEB systems.
ABS_Active_Flag – bool 0/1 CRITICAL Broadcast when ABS modulation is ongoing. ECM receives this to reduce torque; TCM holds current gear; instrument cluster illuminates ABS telltale. Duration and frequency logged for fleet brake analysis.
ESC_Intervention_Flag – bool 0/1 CRITICAL Active during ESC yaw correction. Reduces engine torque and applies asymmetric braking. Extended ESC activity on dry roads indicates suspension wear or tyre mismatch.
Wheel_Slip_Ratio (per wheel) – % -100 – +100 CRITICAL Difference between wheel speed and vehicle speed as a percentage. Negative = locked wheel (braking slip); positive = spinning wheel (drive slip). ABS target slip: 10–20% for peak friction.
HBA_Pressure_Command (per wheel) – bar 0 – 200 DIAGNOSTIC Hydraulic Build/Apply/Hold solenoid commands for each wheel during ABS/ESC. Diagnostic data used to verify modulator solenoid response time and brake circuit functionality.
Rollover_Index – 0–1 0 – 1 CRITICAL Computed metric combining lateral acceleration, roll angle, and load transfer. Above 0.8 threshold triggers RSC (Roll Stability Control) brake application and torque cut in SUVs/trucks.
Brake_Pad_Wear (per axle) – mm or bool 0 – 14 mm MODERATE From resistive wear sensor in pad. Threshold warning at ~2–3 mm remaining. In predictive maintenance systems, wear rate combined with braking frequency gives remaining pad life estimate.

4. Airbag / Supplemental Restraint System Module (ACM / SRS)

PASSIVE SAFETY
The ACM continuously monitors crash sensors and seatbelt status. It makes deploy/no-deploy decisions within 15–50 ms of impact. Operates on a dedicated fault-tolerant CAN bus with redundant power supply. All signals are safety-critical by definition.
Signal Name Unit Typical Range Criticality Significance
Crash_Sensor_Decel (Front/Side/Rear) g 0 – 50+ g CRITICAL MEMS accelerometers in ACM and satellite sensors at B-pillar, door, and front rail. Deceleration profiles are pattern-matched. Frontal crash threshold: typically >8–15 g sustained for 15 ms. Side impact: >20 g for 10 ms.
Seatbelt_Buckle_Status (per seat) bool 0/1 CRITICAL Hall-effect buckle sensor. Determines airbag deployment suppression (unbelted = more aggressive deployment). Also activates pre-tensioner pyrotechnics if crash confirmed. Used for seatbelt reminder chime logic.
Occupant_Detection (Front Passenger) kg / enum 0–120 kg CRITICAL Seat mat pressure sensor array. Classifies occupant as: empty, child seat (out-of-position), small adult, large adult. Determines if front passenger airbag is suppressed (FMVSS 208 child protection requirement).
Airbag_Deployment_Status (per bag) enum armed/deployed/fault CRITICAL Post-crash state of each airbag circuit. Deployed flag is used by eCall/AACN systems (Automatic Collision Notification) to trigger emergency services contact and unlock doors.
Squib_Resistance (per channel) Ω 1.5 – 4.0 Ω CRITICAL Continuous resistance monitoring of each airbag initiator. Out-of-range resistance (<1 Ω or >6 Ω) sets SRS DTC and illuminates airbag lamp. Indicates open circuit, short, or deteriorated squib.
Pretensioner_Fire_Command bool 0/1 CRITICAL Command to fire pyrotechnic seatbelt pretensioners within 5–15 ms of crash detection. Pretensioners remove belt slack before occupant forward motion begins, reducing load limiter force.
Rollover_Sensor_Signal °/s, ° – CRITICAL Angular rate and angle for roll-axis. Triggers curtain airbag deployment in rollover scenarios. Threshold: >60°/s angular rate plus roll angle >75°. Prevents curtain inflation in normal cornering.
eCall_Crash_Flag bool 0/1 CRITICAL Broadcast to TCU (Telematics) when SRS confirms crash with airbag deployment. Triggers eCall (EU regulation 2015/758) — autonomous emergency call to PSAP with vehicle location and crash data.

5. Body Control Module (BCM)

BODY & COMFORT
The BCM is the central hub for body electrical functions. It communicates with low-speed LIN slaves (window motors, seat heaters, mirror adjusters) and higher-speed CAN for chassis integration. It manages power distribution, lighting, locks, and driver personalization.
Signal Name Unit Typical Range Criticality Significance
Battery_Voltage (12V) V 9 – 16 V CRITICAL System supply voltage monitored continuously. Below 11 V triggers load-shedding (disables non-essential consumers). Below 9 V causes ECU resets and CAN bus dropout. Charging voltage 13.8–14.4 V nominal.
Battery_Current (12V) A -400 – +200 A MODERATE From IBS (Intelligent Battery Sensor) on negative terminal. Positive = charging; negative = discharge. Used for SoC estimation, auto start-stop inhibit under low SoC, and parasitic draw diagnosis.
Battery_SoC_12V % 0 – 100 MODERATE Estimated state of charge of 12V lead-acid or AGM battery. Below 80% inhibits start-stop; below 50% triggers charge priority mode; below 20% issues low battery warning to driver.
Ignition_Status enum OFF/ACC/ON/START CRITICAL Master power state broadcast to all ECUs. Drives wake-up sequencing (CAN bus wake), ECU initialization, and pre-drive checks. Critical for correct ECU power mode transitions.
Door_Status (per door) bool 0/1 MODERATE Ajar status from each door latch microswitch. Triggers door-open chime, interior lighting, and prevents auto door locking while open. Broadcast to instrument cluster for door telltale display.
Lock_Status (per door) enum locked/unlocked/latch MODERATE Central locking actuator state. BCM coordinates lock/unlock sequences, child safety lock, and speed-dependent auto-locking. Critical for stolen vehicle immobilization commands via telematics.
Hood_Status bool 0/1 MODERATE Hood open/closed switch. In ADAS vehicles, hood open state may disable pedestrian detection (camera field obstructed) or trigger limp mode in BEV where HV components are accessible.
Exterior_Light_Status (per circuit) bool/PWM 0/1 or 0–100% MODERATE Headlamp, DRL, tail, stop, turn, fog, reverse states. BCM monitors bulb current for open-circuit detection. Matrix LED headlamps: per-pixel PWM duty cycle for ADB (Adaptive Driving Beam) glare control.
Rain_Sensor_Intensity count or % 0 – 100 LOW Optical sensor on windshield measuring light refraction from water droplets. BCM maps to wiper speed intervals. Also used by ADAS to reduce camera reliance in rain and by HVAC to activate auto-defrost.
Ambient_Light_Level lux 0 – 100000 LOW Photodiode on dashboard. Controls automatic headlamp activation, instrument dimming, and HUD brightness. Also input to driver monitoring system for glare compensation.
Window_Position (per window) mm or % 0 – 100% LOW Hall-effect encoder on window regulator motor. Used for auto-close on rain detection, anti-pinch logic (force/position monitoring), and remote window open/close via telematics.
Interior_Temperature °C -20 – 60 LOW NTC thermistor in instrument panel aspirated by fan. Input to HVAC automatic mode. Also used by telematics for cabin pre-conditioning commands in EVs.
Fuel_Level L or % 0 – 100% MODERATE Resistive float sender in fuel tank. Typical sending unit: 0–180 Ω. BCM converts to volume using tank shape lookup table. Low-fuel DTE (Distance-to-Empty) calculated by combining fuel level with average consumption from ECM.

6. Electric Power Steering Module (EPS / PSCM)

CHASSIS
The EPS module drives a brushless DC motor on the steering column or rack. It receives torque sensor input and provides speed-dependent assist. In ADAS-equipped vehicles it also accepts lane-keeping and parking torque overlay commands.
Signal Name Unit Typical Range Criticality Significance

8. Instrument Cluster Module (ICM / Kombi)

HMI
The instrument cluster aggregates signals from the CAN gateway and renders them for the driver. Modern digital clusters (virtual cockpit) run Linux/QNX and display real-time telemetry. The ICM also drives warning lamp logic and odometer.
Signal Name Unit Range Criticality Significance
Speedometer_Display km/h or mph 0–300 MODERATE Sourced from ABS wheel speed. Legal tolerance: +0/-0% (EU) — speedometer must never under-read. Filter applied to prevent rapid needle oscillation from ABS modulation.

CONNECTIVITY
The TCU is the vehicle's connection to cloud infrastructure. It aggregates CAN data, adds GPS position, and transmits over cellular (4G/5G). It supports OTA updates, remote commands, eCall, stolen vehicle tracking, and fleet telematics. Stores a data buffer when connectivity is lost.

10. Battery Management System (BMS) — EV / PHEV / HEV

HIGH VOLTAGE
The BMS is critical to EV/hybrid safety and range. It monitors every cell in the HV pack (400V–800V systems), manages thermal balance, enforces charge/discharge limits, and interfaces with the on-board charger and DC fast charger. Communicates on isolated CAN with the MCU and powertrain bus.
Signal Name Unit Typical Range Criticality Significance
HV_Pack_Voltage V 250–900 V CRITICAL Total HV battery voltage. Continuously monitored for over/under-voltage. NMC cells: 3.0–4.2 V/cell; LFP: 2.5–3.65 V/cell. Outside limits triggers main contactor opening and vehicle shutdown.
HV_Pack_Current A -800 – +800 A CRITICAL From precision shunt or Hall-effect sensor. Positive = discharge; negative = regen/charging. Overcurrent protection limits peak discharge. Integral gives Coulomb-counted SoC.
State_of_Charge (SoC) % 0–100 CRITICAL Estimated usable capacity relative to full charge. Derived from OCV, Coulomb counting, and Kalman filter. Drives range display and charge strategy. Usable window typically 10–90% to preserve cell life.
State_of_Health (SoH) % 0–100 MODERATE Remaining capacity vs. original rated capacity. Below 80% = end of warranted life. Computed from impedance spectroscopy, capacity fade measurement, and cycle history. Drives warranty claims and second-life assessment.
State_of_Power (SoP_discharge / SoP_charge) kW 0–350 kW CRITICAL Available power in next time window (typically 2s, 10s, 30s horizon). Limits motor torque (discharge SoP) and regen torque (charge SoP) to prevent cell damage. Temperature and SoC dependent.
Cell_Voltage (per cell) V 2.5 – 4.25 V CRITICAL Individual cell monitoring via CAN-connected cell monitoring ICs (e.g., LTC6811). Cell delta (>50 mV spread) indicates imbalance or degraded cell. Used to trigger passive balancing discharge through balancing resistors.
Cell_Temperature (per module) °C -40 – 60 CRITICAL NTC thermistors distributed through pack. BMS controls liquid coolant/heat pump to maintain 20–35°C optimal range. Below -10°C restricts charge rate to prevent lithium plating. Above 45°C restricts discharge rate.
Coolant_Inlet_Temp (Battery) °C -10 – 60 CRITICAL Temperature of coolant entering battery pack. BMS uses this with cell temperatures to control chiller valve, heat pump, and coolant pump speed for precise thermal management.
Insulation_Resistance kΩ 0 – 10000 CRITICAL ISO 6469-4 requirement. BMS continuously measures HV-to-chassis isolation via IMD (Insulation Monitoring Device). Below 100 Ω/V (e.g., <40 kΩ on 400V system) triggers fault and may open contactors. Prevents electrocution risk.
Contactor_State (Main+/Main-/Pre-charge) enum open/closed/welded CRITICAL Main HV contactors connect pack to powertrain bus. Pre-charge contactor limits inrush current to inverter capacitors. Welded contactor detection (voltage present after open command) is critical safety check.
Pack_Internal_Resistance mΩ 5 – 200 MODERATE Rising internal resistance indicates aging, cold temperature, or cell degradation. Used to calculate voltage sag under load and update power limit tables. Key input to second-life valuation.
Charge_Rate_Limit (C-rate) C / kW 0 – 3C CRITICAL Maximum allowed charge current from BMS to charger. Varies with temperature, SoC, and SoH. BMS communicates to CCS/CHAdeMO charger via CAN/PLC. At low SoC/temperature: reduced to prevent Li plating.

11. Motor / Inverter Control Unit (MCU / VCU)

EV DRIVETRAIN
The MCU drives the traction motor(s) via a 3-phase inverter. It implements field-oriented control (FOC) or direct torque control (DTC) at microsecond loop rates. The Vehicle Control Unit (VCU) sits above it, arbitrating torque between MCU and BMS limits.
Signal Name Unit Typical Range Criticality Significance
Motor_Speed rpm 0 – 20000 CRITICAL Resolver or encoder feedback on motor shaft. Foundation for FOC current control. Also used to calculate vehicle speed (with gear ratio) and regen availability. Overspeed triggers torque cut.
Motor_Torque_Actual Nm -500 – +500 CRITICAL Estimated actual torque from d/q axis current model. Positive = drive; negative = regeneration. Broadcast to BMS for SoP validation and to VCU for traction control.
Motor_Torque_Command Nm -500 – +500 CRITICAL Driver demand after VCU arbitration (pedal → requested torque → clamped by BMS SoP and MCU thermal limits). Deviation between command and actual >10 Nm for >100 ms triggers fault.
Inverter_Temperature (IGBT/SiC) °C -40 – 150 CRITICAL Junction temperature of power transistors (IGBT or SiC MOSFET). Derating begins at ~100°C; shutdown at 150°C. SiC inverters enable higher switching frequencies and smaller form factors at higher temperatures.
Motor_Winding_Temperature °C -40 – 180 CRITICAL PTC or NTC sensor in stator winding. Above 150°C for PMSM = permanent magnet demagnetization risk. Thermal model updates torque derating maps in real time.
DC_Link_Voltage (Inverter Bus) V 200–900 CRITICAL Voltage on inverter input capacitor bank. Must be within ±10% of nominal for stable switching. Sag during peak demand indicates weak HV bus or contactor resistance. Monitored by MCU independently of BMS.
Phase_Current (Ia, Ib, Ic) A peak 0 – 1200 A CRITICAL Three-phase motor current measured by shunts or Hall sensors. Transformed to d-q frame for FOC. Current imbalance between phases indicates motor winding fault or inverter switch failure.
Regen_Torque_Available Nm 0–500 CRITICAL BMS-limited maximum regenerative braking torque. Communicated to brake blending controller (BBW) so it can fill the difference with hydraulic braking for consistent pedal feel.
Efficiency_Map_Operating_Point % 80–98% LOW Real-time motor efficiency at current speed-torque operating point. Used by VCU for optimal torque split on dual-motor AWD vehicles and for energy consumption reporting.

AUTOMATED DRIVING

1. Fundamentals of Vehicle Diagnostics

1.1 OBD-II Architecture

On-Board Diagnostics II (OBD-II) has been mandatory on all passenger vehicles sold in the US since 1996 and the EU since 2001. It exposes a standardised 16-pin DLC (Data Link Connector) over which a scan tool or telematics device reads:

  • PIDs (Parameter IDs): Real-time sensor values (Mode 01 / 02)
  • DTCs (Diagnostic Trouble Codes): Stored fault codes (Mode 03 / 07 / 0A)
  • Freeze-frame data: Snapshot of sensor values at the moment a DTC was set (Mode 02)
  • Oxygen sensor tests: Mode 05
  • EVAP / readiness monitors: Mode 06 / 01 PID 01
  • VIN & calibration IDs: Mode 09

Appendix A — Freeze-Frame Diagnostics

When a DTC is stored, Mode 02 captures a freeze frame — a snapshot of key PIDs at the fault moment. Always retrieve freeze-frame data alongside the DTC. Critical parameters to always check:

  • Engine RPM and Load at fault event
  • Coolant temperature (was engine warm or cold?)
  • Fuel trims (lean or rich at the moment?)
  • Vehicle speed (was vehicle moving?)
  • Ignition timing advance value
  • MAP / MAF values at fault moment

Appendix B — Fuel Trim Diagnostic Matrix

STFT LTFT Interpretation
Positive (+) Positive (+) System running lean — vacuum leak, MAF under-reading, low fuel pressure, clogged injector

Metrics & KPIs

Utilization & Operations

  • Fleet Utilization Rate — % of scheduled hours actively driven vs. idle or parked
  • Vehicle Uptime / Downtime — hours in operation vs. off-road
  • Geofence Compliance Rate — % of trips within approved zones
  • Route Adherence Score — deviation from planned route in distance or time
  • After-Hours Usage Alerts — ignition-on events outside working hours
  • Trip Count per Vehicle/Day — volume segmented by vehicle and driver
  • Stops & Dwell Time — duration parked at customer, depot, or unauthorized locations

Fuel & Energy

  • Fuel Consumption Rate (L/100km) — from MAF + speed or CAN fuel data
  • Total Fuel Spend by Vehicle / Driver — consumed volume × cost per unit
  • Idle Fuel Waste (L) — fuel burned while RPM > 0 and speed = 0
  • Idle Time % — idle minutes as % of total engine-on time
  • Excessive Idling Events — idle sessions exceeding threshold (e.g., >5 min)
  • Fuel Theft Detection — anomalous fuel level drops without fill events

Driver Behavior

  • Harsh Acceleration Events — throttle % change rate exceeding threshold
  • Harsh Braking Events — deceleration exceeding threshold (e.g., >0.4g)
  • Speeding Events — speed > limit, with duration and severity
